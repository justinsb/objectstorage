# 2026-07-06: S3 frontend (V2)

Added an S3 API frontend (`pkg/s3`) over the same store, after deciding to
defer the GCS gRPC API: gRPC's payoff is Direct Connectivity inside GCP
(irrelevant on a LAN), all official GCS clients default to JSON/HTTP, and
S3 unlocks the self-hosted ecosystem (rclone, restic, kopia, …) — the actual
killer feature for a NAS.

## Decisions

**Separate port instead of protocol sniffing.** S3's `GET /{bucket}/{key}`
collides with the GCS direct-download path; distinguishing them per-request
by headers is fragile. The binary now serves GCS on `--listen` (:8080) and
S3 on `--s3-listen` (:8081), MinIO-style. Same store underneath — the e2e
interop test writes via one protocol, reads via the other, and confirms
identical content written through both protocols shares a single CAS blob.

**aws-chunked decoding is mandatory.** Modern AWS SDKs and the AWS CLI
default to flexible checksums, which wrap PutObject/UploadPart bodies in
`aws-chunked` framing (`x-amz-content-sha256: STREAMING-*`) with trailing
checksum headers. `pkg/s3/awschunked.go` decodes the framing and ignores
chunk signatures/trailers; without it, stored objects would contain the raw
framing bytes.

**No signature verification.** SigV4 headers are accepted and ignored
(trusted-LAN assumption, same as the GCS side). Any credentials work.

**ETag = content MD5, even for multipart.** Real S3 multipart ETags are
`md5-of-part-md5s-<n>`; we return the whole-object MD5 instead, which is
strictly friendlier to integrity-checking clients. Deviation noted in the
README.

**Multipart parts staged as temp files.** Parts land in the bucket's `tmp/`
dir (`store.TempFile`), hashed (md5 → part ETag) as they stream in; Complete
concatenates them through a BlobWriter into the CAS. Sessions are in-memory,
like GCS resumable uploads — same restart caveat, same future `tmp/` sweep
fix.

**ListObjects V1 marker quirk.** V2 continuation tokens map directly onto
the store's inclusive PageToken. V1's `NextMarker` can be a common prefix;
resuming with `marker=<prefix>` must skip the entire group
(`prefixSuccessor`), or the prefix would be re-emitted on the next page.

**S3 conditional requests come almost free.** `If-Match`/`If-None-Match` on
GET/HEAD are handled by `http.ServeContent` against our MD5 ETag;
`If-None-Match: *` on PUT maps to the store's `GenerationMatch=0`
(create-if-not-exists).

## Verification

- 8 new e2e tests against the official aws-sdk-go-v2 client (round-trip
  with metadata/ETag checks, range gets, ListObjectsV2 + paginator,
  12MiB multipart via manager.Uploader, cross-bucket copy, batch delete,
  bucket lifecycle, GCS↔S3 interop + cross-protocol dedup). 17 e2e tests
  total, all passing.
- Real AWS CLI run against a local server: `s3 mb/cp/ls/rm/rb`, including a
  12MiB `cp` (multipart + aws-chunked) and recursive delete
  (ListObjectsV2 `encoding-type=url` + DeleteObjects). Byte-identical
  round-trips.
